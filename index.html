About Network-Probe.

This is an open source project for testing the network connection between any two computers, based on Winsock2, Socket Library.

Versions

The project contains three different versions:

  1. Basic Prototype for network speed probe.
  2. The basic prototype can only run on Windows OS, and it can connect to any host. But the UDP error checking mechanism has some bugs which results in extremely high error rate.

  3. It also contains a hostname resolution function. You can check every website or host's ip address and related information.

  4. Network Probe for multiuser.

  5. You can run only server at a public computer and test the network speed from any of your client computer to the server. No limitation for the client number.
  6. Robustness has been considered and the server runs in multithreading mode. Once a thread died or unexpectedly crash, the other thread will not be affected.

  7. Available for Windows, Ubuntu, and CentOS.

  8. Simple Webserver Probe.

  9. Put all the files that you want to let user access in the same directory. It can handle basic browser HTTP REQUEST, and return the information it want.
  10. Tested with httperf so the robustness can be guaranteed.
  11. Two modes supported: On-Demand Mode and Thread Pool Mode.
  12. Available for Windows, Ubuntu, and CentOS.

How to install?

  • In Linux
$ make
$ ./NetProbe    #will show you the usage 
$ ./NetProbe (arguments according to the usage)
  • In Windows NetProbe.exe to be released. You can compile by yourself. Please select Win32 Console and add EZ_TIMER.H to the header files.
